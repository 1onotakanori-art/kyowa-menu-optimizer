# Kyowa Menu AI - 機械学習環境

## 目的
過去の選択履歴から、ユーザーが選択する可能性の高いメニューを予測するAIモデルを構築します。

## データ構造

### 入力データ（全メニュー）
- パス: `../menus/menus_YYYY-MM-DD.json`
- 構造:
  ```json
  {
    "dateLabel": "1/23(金)",
    "count": 41,
    "menus": [
      {
        "name": "メニュー名",
        "nutrition": { ... }
      }
    ]
  }
  ```

### 教師データ（選択されたメニュー）
- パス: `../../kyowa-menu-history/kyowa-menu-history/data/history/YYYY-MM-DD.json`
- 構造:
  ```json
  {
    "date": "2026-01-23",
    "eaten": ["メニュー1", "メニュー2"],
    "available": ["全メニューリスト"],
    "nutrition": { ... }
  }
  ```

## セットアップ手順

### ステップ1: データ収集 ✅
- kyowa-menu-historyリポジトリから履歴データを取得
- 全メニューデータ（menus/）と選択履歴データ（history/）を確認

### ステップ2: データ統合スクリプト作成（次）
- 全メニューと選択履歴をマージ
- 機械学習用のCSV/JSON形式に変換

### ステップ3: 特徴量エンジニアリング
- メニューの特徴量抽出（栄養素、カテゴリなど）
- 時系列特徴（曜日、過去の選択パターンなど）

### ステップ4: モデル構築
- アルゴリズム選択（ランダムフォレスト、ニューラルネットなど）
- 学習・評価

### ステップ5: モデル統合
- Webアプリへの組み込み
- リアルタイム推薦機能
